Playground App

  • Description
  • Takeaways
  • Technologies Used

After my internship as an Android Engineer with Lowes, I knew I needed to create something of my own in order to solidify my learning. One of the most interesting things I learned was the power of modularization in a massive app. I used this as my starting point. Utilizing Dagger2 I created a multi-module app. The idea is to have each module be it's own section, completely independent and safe from mess-ups in other modules. With this baseline, I employed other design-techniques within the modules themselves.

So far I've used popular libraries such as Moshi/Retrofit and Glide to play around with some APIs I thought were cool. I plan on making new modules for new technologies. I also want to document as much as I can so that I can revisit the code behind the app anytime and refresh myself on previous technologies / patterns. Feel free to check it out!

Compared to the first app I made fueled by Youtube tutorials and a dream, this app is much cleaner. I'm still proud of the first app I did but the design of it is really suboptimal. With my Playground app I use known and popular design patterns. While working on it I can definetly see how much of a benefit solid design is to the dev.

Virtual McSlap 2.0

  • Description
  • Takeaways
  • Technologies Used

Virtual McSlap 1.0 was my first ever personal project. I played on my college's League of Legends team. It's a competitive E-sport where we face off against other schools for prize money. We had a manager (screenname McSlap) who would schedule practice matches against other teams for us. Virtual McSlap 1.0 was just a schedule bot. You can tell it when you booked a match for and it'll be ready to show your schedule on command. This was my first time using Javascript and was once again fueled by Youtube tutorials. I got all the functionality I wanted, albeit with a messy implementation.

Between the time period where my classes ended and my internship began, I decided to give Virtual McSlap another crack. Virtual McSlap 2.0 does everything that 1.0 does except now it can automate a lot of the communication between team managers. I got permission to host it in a popular collegiate channel. I took a class in Javascript web programming between 1.0 and 2.0 and was able to get everything I wanted in a quarter of the time. Where 1.0 used local storage, 2.0 uses Firestore. I use Amazon Web Services to host the bot.

I learned a lot about myself while working on 2.0. For one, if I'm working on something I'm passionate about and interested in I become a bit of a workhorse. I stayed up later than I probably should have for more nights than I can remember. This was right after school ended and I had more fun working on this bot than anything else I could be doing in the Summer. Secondly, revisiting and reworking projects provide such fulfilling experience. It's night and day between 1.0 and 2.0 and I'm proud of how much I've grown in the year I had between them.

Simple Finance Log App

  • Description
  • Takeaways
  • Technologies Used

Simple Finance Log was my first attempt at any sort of Android development. I wanted a way to track my spending. I also needed to keep track of any money friends owed me from going out for food/groceries/etc. I felt like this is just the type of thing to let software do. And what could be more convenient than something I could put on my phone? Simple Finance Log did everything I set out to accomplish but in a very crude fashion. I learned a lot about basic Android development with this app.

Simple Finance Log allows you to log purchases with a category, price, and date. You can then view statistics about what you're spending on your money on, how much you spent each day/week/month. There's definetely a lot of design work to be done. I put it up on the Google Play Store for fun once I was done.

Simple Finance Log is something that I would like to label as a "beginner's passion project." I saw a need and wanted to create something cool. I thought to myself: "Even if nobody else uses this, I'll use it myself." I did this in the Summer of my Sophomore year, just before I got into any sort of front-end programming courses. This was my first interaction with HTML/CSS and any design like that. I'm certain I Googled how to center this or that at least 20 times over the course of development.

One thing I didn't expect out of this project was the discovery of how much I love working on things I actually like. Up until this point, classes have always been something along the lines of "Sort this array" or "Use this data structure to implement this console program" or "What's the two's complement of X?" Not to discount the value of all this computer theory, but there's something really rewarding about building real software like an app.
